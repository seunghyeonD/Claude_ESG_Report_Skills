## 2. 데이터 정합성 (Data Consistency)

### 2.1 수치 교차 검증

| 체크 항목 | 검증 공식 | 허용 오차 |
|----------|----------|----------|
| 재생에너지 비율 | = 재생에너지 / 총에너지 × 100 | ±0.1%p |
| 폐기물 재활용률 | = 재활용량 / 총발생량 × 100 | ±0.1%p |
| 여성 비율 | = 여성 인원 / 전체 인원 × 100 | ±0.1%p |
| 사외이사 비율 | = 사외이사 / 전체이사 × 100 | 정확 |
| GHG 원단위 | = (Scope 1+2) / 매출액 | ±0.01 |
| 에너지 원단위 | = 총에너지 / 매출액 | ±0.01 |
| Scope 2 관계 | Market-based ≤ Location-based (일반적) | 예외 가능 |
| 임직원 합계 | = 정규직 + 비정규직 | 정확 |
| 이사회 합계 | = 사내이사 + 사외이사 + 기타 | 정확 |

### 2.2 트렌드 합리성

| 체크 항목 | 플래그 기준 | 대응 |
|----------|-----------|------|
| 전년 대비 ±50% 이상 변동 | 모든 주요 KPI | 변동 사유 서술 필수 |
| 전년 대비 ±30% 이상 변동 | GHG, 에너지, 매출 | 사유 확인 권장 |
| 역방향 트렌드 | 목표 대비 악화 추세 | 원인 분석 및 대응계획 서술 |
| 데이터 불연속 | 보고 경계 변경, 방법론 변경 | 재작성(restatement) 설명 |

### 2.3 단위 일관성

| 체크 항목 |
|----------|
| GHG 단위: tCO2e 통일 |
| 에너지 단위: TJ 또는 MWh 통일 (혼용 금지) |
| 수자원 단위: m³ 통일 |
| 폐기물 단위: 톤 통일 |
| 금액 단위: 백만원 통일 (본문), 억원 병기 허용 |
| 비율: % 통일, 소수점 1자리 |
| 원단위: 분모 단위 명확 (매출 기준 vs 생산량 기준) |

## 6. 구조 및 형식 (Structure & Format)

### Markdown 검증:
| 체크 항목 |
|----------|
| 목차(TOC) 완성 및 앵커 링크 동작 |
| 표(table) 렌더링 정상 |
| 제목 계층구조 일관 (h1 > h2 > h3) |
| 프레임워크 배지 표기 일관 (`GRI 305-1` 형식) |
| 누락 데이터 표기 일관 (`[데이터 수집 필요 / Data to be collected]`) |
| 추정치 표기 일관 (`[추정 / Estimated]`) |
| 각주/참고 표기 |

### HTML 검증:
| 체크 항목 |
|----------|
| 브라우저에서 정상 표시 |
| 인쇄(Print) 레이아웃 정상 |
| 언어 토글 동작 (KR/EN/Both) |
| 컬러 코딩 적용 (E=초록, S=파랑, G=보라) |
| 표 스크롤/반응형 정상 |
| TOC 네비게이션 앵커 링크 동작 |
| 외부 의존성 없음 (self-contained) |
| 폰트 가독성 (한/영 모두) |

## 검증 실행 순서

1. **구조 확인** — 목차 vs 본문 매칭, 빈 섹션 없는지
2. **데이터 정합성** — 수치 교차 검증, 단위 통일
3. **프레임워크 완전성** — 부록 색인 테이블 완성도
4. **중대성 커버리지** — 중대 주제 ↔ 본문 매칭
5. **이중언어** — 빠진 번역 없는지
6. **보증 적격성** — 방법론, 경계, 추정치 표시
7. **규제 준수** — 의무 공시 항목 누락 없는지
8. **HTML 렌더링** — 시각적 검증
9. **면책 조항** — 포함 확인
